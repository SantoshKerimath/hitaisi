# HITAISI – Product Document

## Vision

Modular digital health benefits infrastructure platform.

## Users

- Platform Admin
- Insurer Ops
- Broker
- HR Admin
- Employee
- Dependents
- Claims Ops (future)

## Core Domains

1. Identity
2. Clients
3. Products
4. Policies
5. Members
6. Premium Engine
7. Ledger / Buffer
8. Tickets
9. Audit
10. Claims

## Key Business Rules

- Premium derived from demographic matrix
- Buffer auto-updates on member add/delete
- Cover dates independent from policy dates
- Audit auto-tracked via signals
- JWT secured endpoints

## Roadmap

Phase 1: Core benefits engine
Phase 2: Claims + TPA
Phase 3: Payment gateway
Phase 4: Multi-tenant SaaS deployment
